Loading IgnitedRecord

To use IgnitedRecord, you have to load it first.
You can choose four different approaches to loading IgnitedRecord:

Load as a Model

Before you load any of the derived models/classes, use Loader::model() to load IgnitedRecord.

$this->load->model('ignitedrecord/ignitedrecord');
$this->load->model('some_other_model');

Load with the Custom Loader

$this->load->orm();

This also works as a wrapper for Loader::model(), which enables you to do this:

$this->load->orm('some_other_model');

Then you have loaded both IgnitedRecord and some_other_model at the same time!

Autoload

Just add 'ignitedrecord/ignitedrecord' in the congfig/autoload.php's model array:

/*
| -------------------------------------------------------------------
|  Auto-load Models
| -------------------------------------------------------------------
| Prototype:
|
|    $autoload['model'] = array('model1', 'model2');
|
*/

$autoload['model'] = array('ignitedrecord/ignitedrecord');

And don't forget to autoload the database library.

PHP include()

This is the least recommended of these solutions, just use the PHP function include():

include APPPATH.'models/ignitedrecord/ignitedrecord.php';

I recommend using the APPPATH constant, then everything is relative to the app root.
