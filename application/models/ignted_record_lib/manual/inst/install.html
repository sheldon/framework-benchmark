How To Install IgnitedRecord

Important:  If you use PHP 4 you must also read the PHP 4 specific instructions, because there are a few differences between PHP 4 and PHP 5.

Basic Installation

Drop the ignitedrecord folder into your models dir, and you're ready to go!

IgnitedQuery

IgnitedQuery is a SQL string builder, like CodeIgniter's ActiveRecord implementation. It has support for building multiple queries at the same time (CI's AR only allows one at a time) through multiple instances. It also has support for nested WHERE statements and subqueries.

Optional, but makes it easier when fetching a lot of objects (no need to worry about queries interfering with eachother) and provides a bit more functionality.

The custom Loader

The custom loader is a class which extends the CI_Loader and makes it easier to instantiate and load IgnitedRecord.
Is is an optional component, and not needed for normal use. But it simplifies things a bit.

YAYparser

YAYparser is a YAML parser which IgnitedRecord uses to parse yaml table definitions (which stores the data instead of pure PHP).
It is used by yamlfactory() which makes it optional, unless you want to be able to save table definitions in YAML files.

Complete Installation - Step by Step

  1. Drop the folder application/models/ignitedrecord into your application/models dir.
  2. If you want IgnitedQuery:

    Drop the ignitedquery.php file into application/libraries.

  3. If you want the custom loader:
    • Drop the MY_Loader.php file into application/libraries.
    • If you already have a custom MY_Loader.php, or if you have a custom Loader.php you should copy the orm() method into that loader.
  4. If you plan to use yamlfactory(), copy yayparser_helper.php into your application/helpers dir.
  5. If you use PHP 4, read the PHP 4 specific instructions.
