Manipulating Data

Data is stored in IR_record objects, whose properties store the data of the row.
Here is an example of the data in an IR_record:

IR_record Object (
    [__table] => string(7) "threads"
    [__id] => string(1) "1"
    [id] => string(1) "1"
    [name] => string(10) "new thread"
    [data] => string(13) "hello comment"
)

The __table and __id are properties that link that record to the right row in the database.
There is also a few other "hidden" properties (prefixed by __) that are not displayed here, because they take too much space.

Use the Customizable print_r() to avoid being buried in data when you dump your objects.

Well, back to the point...
The row data is stored as properties on IR_record objects. To edit or read the data, you only have to access the values of their properties:

$rec = $this->user->find_by('name', 'Martin');
$rec->name = $rec->name . ' Wernstahl';
echo $rec->name; // echoes "Martin Wernstahl"

uid()

This method of the IR_record object provides the unique identifier of the row wich the record is mapped to.
It is usually an integer (and in the case above, it was 1).

If it is not mapped to a record, false is returned.

in_db()

Also an IR_record method, this returns true if it is mapped to a database row, false otherwise.

load_data(data)

This method populates the IR_record object with the data in the data array.

Example:

$rec = $this->user->get(); // I'll edit a user, I don't care who it is :P
echo $rec->name; // well, it echoes something...
$rec->load_data(array('name' => 'Foo Bar', 'email' => 'foo.bar@example.com'));
echo $rec->name; // echoes 'Foo Bar'

get_data()

The opposite of load_data(), returns the row data in an array.

Example:

$rec = $this->user->get();
$data = $rec->get_data();

foreach($data as $column => $value)
{
    echo "$column => $value\n";
}
