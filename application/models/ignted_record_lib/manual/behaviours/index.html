Using act_as

What is a behaviour?

A behaviour is a class which provides extra functionality to IgnitedRecord.
Behaviours can help with tree / list traversal, timestamps of entries, revisions and much more.

How do they integrate with IgnitedRecord?

They are instantiated with the IgnitedRecord instance and the options provided as first and scond parameter. Then they are assigned to a property with the same name as the behaviour name.

After that their constructor attaches their code to the hooks and __call() takes care of the aggregation (aggregation only works under PHP 5).

They can also add helper classes to the record objects, which act like "mini behaviours" for the records.

Installing a behaviour

To install a behaviour, simply drop the behaviour's behaviourname.php file in application/models/ignitedrecord/behaviours.
The behaviour may depend on other files, so you must also check their documentation.

Loading them

The property $act_as specifies which behaviours IgnitedRecord should use, you can also use the act_as() method (which can load behaviours on-the-fly).
You can specify behaviour names in a string (when you're using a single behaviour) or in an array:

class Page extends IgnitedRecord{
    var $act_as = 'tree';
}

The code above tells IgnitedRecord to create a model mapped to the table pages, and to load the tree behaviour.
Behaviours can be configured, which you do with an associative array passed to act_as:

var $act_as = array('behaviour_name' => array('setting1' => 'value1', 'setting2' => 'value2'));
// the method way:
$this->comment->act_as(array('timestamped' => array('updated_at' => 'changed')));
